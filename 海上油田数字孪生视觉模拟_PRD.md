## 1. 项目背景与定位

### 1.1 背景

海上油田(钻井平台、FPSO、穿梭油轮等)是数字孪生最具视觉冲击力的应用场景之一。传统孪生项目往往"数据先行、视觉将就",导致演示效果平庸、难以打动评委与客户。本项目反其道而行:**一期专注把视觉呈现做到电影级**,数据孪生作为二期能力,通过预留的状态驱动架构平滑接入。

### 1.2 产品定位

- **一期定位**:一个"看起来已经是数字孪生"的高保真海上油田实时 3D 模拟场景,由 mock 状态模拟器驱动,具备完整的动态感与工业氛围感。
- **二期定位**:替换数据源为真实/仿真 SCADA 数据,升级为真正的数字孪生监控平台。
- **一句话描述**:用游戏级渲染技术,构建一座"活着的"海上油田。

### 1.3 目标用户与使用场景

| 用户 | 场景 |
|---|---|
| 项目演示者(本人/团队) | 比赛路演、课程答辩、作品集展示 |
| 评委 / 导师 / 潜在合作方 | 浏览器打开即看,无需安装,巡航镜头自动展示 |
| 二期:油田运维人员(假想) | 设备状态总览、告警定位、点击聚焦查看指标 |

### 1.4 成功标准(一期)

1. 桌面端 Chrome 稳定 60fps,移动端 ≥30fps;
2. 任意时刻截图具备"封面级"质感(可直接用于海报/朋友圈/路演 PPT);
3. 场景具备昼夜、天气、设备状态三类动态变化,演示 3 分钟内不重复;
4. 状态驱动架构跑通:修改 mock 状态 JSON,场景视觉即时响应,零渲染层改动。

## 2. 总体架构

### 2.1 三层架构:渲染层 — 状态层 — 数据适配层

```
┌─────────────────────────────────────────────┐
│  场景渲染层 (React Three Fiber)              │
│  只消费状态,负责"好看"                       │
│  海洋 / 天空 / 平台 / FPSO / 粒子 / 后处理    │
└──────────────────△──────────────────────────┘
                   │ 订阅 (Zustand selector)
┌──────────────────┴──────────────────────────┐
│  状态层 (Zustand Store)                      │
│  devices / environment / camera / timeline   │
│  统一入口: updateDeviceState(id, patch)      │
└──────────────────△──────────────────────────┘
                   │ 调用同一接口
        ┌──────────┴───────────┐
        │                      │
┌───────┴────────┐    ┌────────┴─────────────┐
│ 一期: Mock 模拟器│    │ 二期: WebSocket 网关  │
│ (脚本化/随机)    │    │ (NestJS + SCADA 适配) │
└────────────────┘    └──────────────────────┘
```

**核心原则:场景不读数据,只读状态。** 一期与二期的唯一差异是状态层的"喂数方"不同,渲染层零重构。

### 2.2 技术选型

| 模块 | 选型 | 理由 |
|---|---|---|
| 前端框架 | React 19 + Vite 6 + TypeScript | 团队最熟,与既有项目栈一致 |
| 3D 引擎 | Three.js + React Three Fiber + drei | 声明式场景组织,drei 提供 Ocean/Sky/Html 等现成件 |
| 后处理 | postprocessing 库 | Bloom / SSAO / God Rays / ACES 色调映射 |
| 状态管理 | Zustand | 轻量、selector 订阅、与 R3F 协作成熟 |
| 动画 | R3F useFrame + maath / GSAP(相机运镜) | 帧驱动 + 时间线运镜 |
| UI 浮层 | Tailwind CSS v4 + Framer Motion | HUD、设备标注卡片、控制面板 |
| 模型工具链 | Blender + gltf-transform(Draco/Meshopt 压缩) | 资产瘦身与 LOD 生成 |
| 后端(预留) | NestJS + WebSocket(socket.io) | 一期仅提供 mock 状态模拟器接口 |
| 部署 | Docker + Nginx / Zeabur | 静态托管即可,后端可选 |

### 2.3 校验与类型

- 设备状态 Schema 用 **Zod** 定义,mock 数据与未来真实数据共用同一套校验;
- 状态类型从 Zod schema 推导(`z.infer`),保证前后端契约一致。

## 3. 场景内容设计

### 3.1 场景要素清单

| 要素 | 优先级 | 说明 |
|---|---|---|
| 海面 | P0 | 全场基底,质感第一来源 |
| 天空 / 光照 | P0 | HDRI + 太阳方位驱动,决定整体色调 |
| 主钻井平台(导管架式) | P0 | 场景主角,井架 + 甲板 + 生活楼 + 直升机坪 |
| 后处理管线 | P0 | Bloom / 雾 / 色调映射,电影感的一半来源 |
| 火炬塔燃烧 | P1 | 视觉记忆点,夜景核心 |
| FPSO 浮式储油船 | P1 | 第二主体,随浪起伏 + 系泊缆 |
| 平台细节动效 | P1 | 障碍灯闪烁、吊机摆动、水线浪花 |
| 昼夜循环 | P2 | 太阳/月亮轨迹、灯光自动切换 |
| 天气系统 | P2 | 晴 / 雾 / 雨三态,可切换可过渡 |
| 穿梭油轮 | P2 | 沿航线移动、靠泊 FPSO |
| 直升机起降 | P2 | 路径动画 + 旋翼模糊 |
| 相机巡航系统 | P2 | 预设机位 + 电影运镜 + 点击聚焦 |
| 海底视角(彩蛋) | P3 | 水下散射光、立管延伸至海床 |

### 3.2 P0:基底场景(第 1 周)

**目标:一张截图立住质感。**

#### 3.2.1 海面

- 起步:drei `<Ocean>` / three.js Water shader,快速出效果;
- 进阶:Gerstner 多波叠加(4~8 组波参数)或 FFT 海洋着色器;
- 必备细节:菲涅尔反射、太阳高光拉丝(specular streak)、近岸/构筑物泡沫贴图、波峰白沫(基于波高阈值);
- 海面提供 `getHeightAt(x, z)` 采样接口,供浮体同步起伏(FPSO、油轮复用)。

#### 3.2.2 天空与光照

- HDRI 环境贴图(Poly Haven 海上/黄昏 HDRI,2K 足够)负责 PBR 反射;
- drei `<Sky>` + DirectionalLight 跟随太阳角度,默认机位用**低角度夕阳光**(黄金时刻),逆光 + 轮廓光最出片;
- 环境光遮蔽:烘焙 AO 贴图(Blender)+ 运行时 SSAO 叠加。

#### 3.2.3 主钻井平台

- 来源:Sketchfab CC 授权模型改造,或 Blender 自建简化版(井架桁架 + 甲板模块 + 吊机 + 直升机坪 + 生活楼);
- 材质:PBR 三件套(BaseColor / Metallic-Roughness / Normal),叠加**锈蚀与油污细节贴图**(detail map, 工业感关键);
- 面数预算:主平台 ≤ 150k 三角面(LOD0),压缩后 ≤ 8MB。

#### 3.2.4 后处理管线(postprocessing)

按顺序:`SSAO → Bloom(threshold 0.85, intensity 1.2 起调) → 体积雾/高度雾 → Vignette → ACES ToneMapping`。移动端降级方案见 §6。

### 3.3 P1:工业细节(第 2 周)

#### 3.3.1 火炬燃烧(视觉记忆点)

- 火焰:平面/锥体网格 + 噪声扰动 shader(simplex noise 滚动 UV),双层叠加(内焰亮黄、外焰橙红);
- 烟雾:GPU 粒子(InstancedMesh + 自定义材质),受"风向"状态参数影响飘散方向;
- 光照:火炬顶端挂 PointLight,强度做 flicker 噪声抖动;夜景 + Bloom 下为全场视觉锚点;
- 状态联动:火炬开/关、火焰大小由设备状态 `flare.metrics.intensity` 驱动。

#### 3.3.2 FPSO 浮式储油船

- 船体随浪起伏:取船体下方 4 点海面高度采样,拟合 pitch/roll/heave;
- 系泊缆:CatmullRom 曲线 + TubeGeometry,两端跟随浮体动态更新;
- 甲板细节:储罐、火炬臂、卸油臂、舷灯。

#### 3.3.3 平台细节动效

- 航空障碍灯:红色闪烁(emissive 脉冲),多平台错相;
- 吊机:缓慢回转 ±15°,吊钩轻微摆动(单摆模拟);
- 导管架水线:浪花泡沫精灵片(billboard)+ 透明度随波高变化;
- 远景大气透视:距离雾使远处平台/船只渐隐,拉开空间层次。

### 3.4 P2:动态与氛围(第 3 周)

#### 3.4.1 昼夜循环

- `environment.timeOfDay ∈ [0, 24)` 单一状态驱动:太阳方位角/高度角、天空色、HDRI 强度、灯光开关阈值;
- 关键时段调色预设:黎明(冷蓝)、正午(中性)、黄金时刻(暖橙,默认)、夜晚(深蓝 + 灯火);
- 可加速播放(演示模式 1 分钟 = 24 小时)。

#### 3.4.2 天气系统

| 天气 | 视觉手段 | 状态参数 |
|---|---|---|
| 晴 | 默认 | `weather: "clear"` |
| 雾 | 指数雾密度提升 + 去饱和 + 能见度衰减 | `weather: "fog"` |
| 雨 | 粒子雨幕 + 屏幕水滴 shader + 海面波高提升 + 天空压暗 | `weather: "rain"` |

天气切换做 5~10 秒插值过渡,不允许跳变。

#### 3.4.3 动态实体

- 穿梭油轮:CatmullRom 航线巡航 → 减速 → 靠泊 FPSO(尾输位)→ 停留 → 离泊,状态机驱动;
- 直升机:起飞 → 巡航绕场 → 降落直升机坪,旋翼用纹理旋转 + 运动模糊片模拟;
- 守护船(可选):围绕油田缓慢巡逻,丰富中景。

#### 3.4.4 相机系统

- **自由模式**:OrbitControls(限制最低俯角,禁止穿入海面);
- **巡航模式**:3~5 条预设电影运镜(CatmullRom 路径 + lookAt 目标插值),如"海面低空掠过 → 拉升环绕平台 → 推近火炬";
- **聚焦模式**:点击设备 → GSAP 平滑飞向预设观察位 → 弹出标注卡片;
- 演示一键启动:巡航 + 昼夜加速 + 随机事件,3 分钟无人值守 demo。

### 3.5 P3:海底彩蛋(可选)

相机下潜过水面时切换水下渲染:蓝绿色散射、God Rays 光柱、悬浮颗粒粒子、立管与导管架延伸至海床。作为路演"哇点"保留,不占主线排期。

## 4. 状态驱动设计(孪生预留核心)

### 4.1 设备状态 Schema(Zod)

```ts
const DeviceStatus = z.enum(["running", "warning", "fault", "offline"]);

const DeviceState = z.object({
  id: z.string(),            // 如 "platform-A.crane-1"
  type: z.enum(["platform", "fpso", "flare", "crane",
                "pump", "tanker", "helicopter"]),
  status: DeviceStatus,
  metrics: z.record(z.number()),  // 如 { pressure: 3.2, output: 870 }
  updatedAt: z.number(),
});
```

### 4.2 视觉映射规则

| status | 视觉表现 |
|---|---|
| running | 设备指示灯绿色,动画正常速率 |
| warning | 指示灯黄色脉冲,标注卡片黄边 |
| fault | 指示灯红色急闪 + 局部 Bloom 提亮,可选告警音 |
| offline | 设备材质去饱和压暗,动画停止 |

`metrics` 驱动连续量:火焰大小、泵机动画速率、储罐液位(可视化条)等。

### 4.3 统一事件入口

```ts
// 一期 mock 模拟器与二期 WebSocket 都只调用这一个口
updateDeviceState(id: string, patch: Partial<DeviceState>): void
```

### 4.4 Mock 状态模拟器(一期)

- 运行于前端(可抽到 NestJS),按脚本时间线 + 随机扰动产生状态流;
- 内置"剧情脚本":如第 45 秒注水泵 warning → 第 60 秒升级 fault → 镜头自动聚焦 → 第 90 秒恢复,用于路演讲故事;
- 模拟器与渲染完全解耦,关闭模拟器场景静态可用。

### 4.5 标注层(HUD)

- drei `<Html>` 锚定设备坐标,显示名称 + status + 2~3 个关键 metrics;
- 默认收起为小圆点,hover/点击展开卡片;距离裁剪(远处不渲染 DOM);
- 全局 HUD:左上场景时间/天气,右上设备健康总览(绿黄红计数),底部相机模式切换。

### 4.6 二期接入路径(本期不实现,仅约定)

1. NestJS 网关订阅 SCADA / IoT 平台(MQTT/OPC UA 适配器);
2. 网关将原始遥测翻译为 DeviceState patch,经 WebSocket 推送;
3. 前端将 mock 模拟器替换为 socket 订阅,调用同一 `updateDeviceState`;
4. 渲染层、标注层零改动。

## 5. 资产规范

| 项 | 规范 |
|---|---|
| 模型格式 | glTF 2.0 + Draco 或 Meshopt 压缩 |
| 贴图 | PBR Metallic-Roughness 流;主体 2K,小物件 1K;KTX2/Basis 压缩优先 |
| 总包预算 | 模型 + 贴图 ≤ 20MB(gzip 前),首屏懒加载,远景实体延迟加载 |
| LOD | 主平台/FPSO 各 3 级(150k / 40k / 8k 面),gltf-transform 生成 |
| 命名 | 节点命名与设备 id 对齐(如 `platform-A_crane-1`),便于状态绑定 |
| 授权 | Sketchfab 仅取 CC-BY / CC0,记录署名清单;HDRI 用 Poly Haven(CC0) |

## 6. 性能预算与降级策略

| 指标 | 桌面 | 移动 |
|---|---|---|
| 帧率 | 60fps | ≥30fps |
| Draw calls | ≤ 300 | ≤ 150 |
| 三角面(视锥内) | ≤ 1.5M | ≤ 500k |

手段:
- 重复构件(栏杆、管线、灯具)全部 InstancedMesh;
- 海面网格按相机距离衰减细分(近密远疏);
- 后处理降级:移动端关 SSAO / God Rays,保 Bloom + ToneMapping;
- 粒子量按设备性能分档(检测 GPU tier);
- DOM 标注限制同屏 ≤ 12 个;
- 启动时性能自检,自动选择 高/中/低 画质档,用户可手动覆盖。

## 7. 里程碑计划(4 周)

| 周次 | 阶段 | 交付物 | 验收标准 |
|---|---|---|---|
| W1 | P0 基底 | 海面 + 天空 + 主平台 + 后处理 | 黄金时刻截图达"封面级";60fps |
| W2 | P1 细节 + 状态机制 | 火炬、FPSO、细节动效;状态层 + mock 模拟器跑通 | 改 mock JSON 场景即时响应;夜景火炬出片 |
| W3 | P2 动态氛围 | 昼夜、天气、油轮/直升机、相机巡航 | 3 分钟无人值守 demo 不重复、不跳变 |
| W4 | 打磨发布 | 性能优化、画质分档、部署上线、录制 demo 视频 | 移动端 ≥30fps;线上可访问;视频成片 |

风险与对策:
- **模型资产不达标** → 备选方案:Blender 程序化简模 + 强材质/光照补质感(光照能救 70% 的模型);
- **FFT 海洋调参耗时** → W1 锁定 Gerstner 方案即可达标,FFT 列为可选优化;
- **移动端性能不足** → 降级档保底,路演以桌面端为准。
